


    MVC 기반의  console 회원제 게시판 (CRUD)
        1. 구현할 기능
            회원: 회원가입, 로그인, 아이디찾기, 비밀번호 찾기, 회원탈퇴, 회원수정
            게시판: 글작성, 글전체출력,글개별출력, 글삭제, 글수정, 조회수
            댓글: 댓글작성, 댓글출력, 댓글삭제, 댓글 수정
        2. 데이터베이스 설계
            ---샘플 데이터 이용한 데이터 파악 9머리 안그려지면 눈이 보이도록 시각화 == 협업에는 각 생각들이 다르기 때문에 추상화 -->
                                                                                                            시각화 설계)
            1. 필드 파악 : 저장/영구 할 데이터들을 파악
            아이디 , 비밀번호, 이름, 연락처, 가입일, 글제목,글내용,글작성일,글작성자,글조회수
            댓글내용,댓글작성일 (저장할것)
            -유효성검사[x] = 비밀번호 확인시[x-영구저장안함] / 연산데이터[x]=순위,총합계,평균 (실시간-계속바뀜)

            2. 필드 분류 : 필드들을 기능별로 테이별형식으로 분류 , 중복 최소화 분류 ,로그/기록 따른 분류
                회원기능 : 아이디, 비밀번호 ,이름, 연락처, 가입일
                게시물기능: 글제목,글내용,글작성일,글작성자,글조회수
                댓글기능: 댓글내용,댓글작성일

            3. 필드 관계    : 1:1  , 1:n  , (실무x)n:n
                회원 게시판 관계  : 회원이 게시물을 작성 , 회원 한 명이 한개를 작성하는건지 ,회원한명이 게시물 여러개인지  [1:N]
                회원 댓글 관계 : 회원이 댓글을  작성, 회원1명이 댓글을 1개 다는건지 ,회원1명이 여러개인지               [1:N]
                게시판과 댓글 관계  : 게시물 안에 댓글 작성, 게시물 1개에 댓글1개인지, 게시물1개에 댓글 여러개인지        [1:N]
                *PK -Fk : 1-PK  N-FK
                *관례적으로 테이블 1개당 PK 1개
                회원번호PK ---작성자(회원번호FK)
                회원번호PK ---댓글작성자(회원번호FK)
                글번호PK  ---댓글위치(글번호FK)

                -SQL 작성,샘플

    2. 자바 프로젝트 패키지 구조 설계

    view 패키지                                                역할: 입출력

        memberController 클래스
        BoardConroller 클래스

    model 패키지 : dto,dao 클래스들을 보관하는 패키지              역할: 데이터 관리

       dto패키지
